# DoNotOpenGL

I started learning OpenGL using this [tutorial](https://learnopengl.com/Introduction) to render a music video for a song I'm currently writing. I decided to call the project DoNotOpenGL (lol). The lighting implmentation is in progress.

There is a demo scene called `Scene1` in `src/scenes/scene_1.cpp` you can run using the steps below. The textures are from [here](https://www.solarsystemscope.com/textures/).

## Repository Layout

* `CMakeLists.txt` - CMake build configuration for the `EngineCore` library and the `App` executable.
* `include/` - Public headers for the engine, graphics types, and scenes.
* `src/` - Implementation files for the engine, graphics helpers, GLAD, and scenes.
* `shaders/` - Vertex and fragment shader files copied next to the executable at build time.
* `assets/` - Texture and other runtime assets copied next to the executable at build time.

The `build/` directory is generated by CMake and should not be edited manually.

## GLAD Setup

This repository does not include the generated GLAD source file by default.

Before building, add your own `glad.c` file:

1. Go to the GLAD generator:

   * https://gen.glad.sh/

2. Use these settings:

   * Language: `C/C++`
   * Specification: `OpenGL`
   * API: choose your OpenGL version (for example `OpenGL 3.3`)
   * Profile: `Core`
   * Generate a loader: enabled

3. Generate and download the files.

4. Copy:

   * `src/glad.c` → `src/glad/glad.c`
   * `include/glad/` → your project's include path if not already present

Your structure should look like:

```text
src/
    glad/
        glad.c

include/
    glad/
        glad.h
```

Without `src/glad/glad.c`, linking will fail with missing OpenGL loader symbols.

## stb_image Setup

This project uses `stb_image` for texture loading.

Before building, add `stb_image.h` manually:

1. Download `stb_image.h` from:

   * https://github.com/nothings/stb/blob/master/stb_image.h

2. Place it inside:

```text id="z0u97n"
include/stb/stb_image.h
```

Your structure should look like:

```text id="u4wxca"
include/
    stb/
        stb_image.h
```

## Build And Run

From the repository root:

```bash
mkdir build
cd build
cmake ..
cmake --build .
./App.exe
```

On Windows PowerShell, you can also launch the executable with:

```powershell
.\App.exe
```

## Making Your Own Scene

The scene system is defined by `include/scenes/scene.h`. A scene must implement:

* `init()` - load resources and create scene objects.
* `update(float deltaTime)` - update simulation state each frame.
* `render(Camera &camera, float aspectRatio)` - draw the scene.
* `processInput(Input &input, float deltaTime)` - react to keyboard and mouse input.
* `getCamera()` - return the scene camera.

To create a new scene:

1. Add a new header in `include/scenes/` that inherits from `DoNotOpenGL::Scene`.
2. Add the implementation in `src/scenes/`.
3. Use `application->getResourceManager()` inside `init()` to load shaders and textures.
4. Build your scene objects with the graphics types in `include/graphics/`, such as `Camera`, `Mesh`, `Object`, `Shader`, and `Texture`.
5. Register the new scene source file in `CMakeLists.txt` under the `App` target.
6. In `src/main.cpp`, include your scene header and replace `Scene1` with your new scene:

```cpp
app.setScene(std::make_unique<DoNotOpenGL::YourScene>(&app));
```

### Minimal Scene Skeleton

```cpp
class YourScene : public DoNotOpenGL::Scene {
public:
	YourScene(DoNotOpenGL::Application *app) : Scene(app) {}

	void init() override {}
	void update(float deltaTime) override {}
	void render(DoNotOpenGL::Camera &camera, float aspectRatio) override {}
	void processInput(DoNotOpenGL::Input &input, float deltaTime) override {}
	DoNotOpenGL::Camera &getCamera() override { return camera; }

private:
	DoNotOpenGL::Camera camera;
};
```

## Runtime Notes

* `main.cpp` initializes `DoNotOpenGL::Application`, sets the active scene, and then enters the main loop.
* `CMakeLists.txt` builds `EngineCore` as a static library and links `App` against it.
* Shaders are expected under `shaders/vertex/` and `shaders/fragment/`.
* Asset paths in the code are relative to the executable, because CMake copies `assets/` and `shaders/` into the output directory.
